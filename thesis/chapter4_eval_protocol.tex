\subsection{Протокол оценки}
\label{sec:eval_protocol}

\subsubsection*{Метрики качества}

Применяется набор из пяти метрик, каждая измеряет отдельный аспект качества.

\textbf{DINO-style}~--- косинусное сходство \texttt{[CLS]}-токенов DINO~ViT-B/8~\cite{caron2021dino} между сгенерированным изображением и стилевым референсом. \textit{Основная} метрика: авторы B-LoRA~\cite{frenkel2024blora} используют именно DINO-сходство в Table~1 оригинала, поскольку DINO-признаки чувствительны к текстуре и художественной манере независимо от семантики. Модель \texttt{facebook/dino-vitb8}, среднее по батчу нормализованных эмбеддингов.

\textbf{CLIP-style}~--- косинусное сходство эмбеддингов CLIP~\cite{radford2021clip} (ViT-L/14) сгенерированного изображения и стилевого референса; L2-нормировка. Абсолютные значения CLIP-style в задаче переноса стиля ограничены конструкцией CLIP-пространства, при значениях $\gtrsim 0{,}7$ интерпретация осторожна.

\textbf{CLIP-content}~--- косинусное сходство эмбеддинга CLIP сгенерированного изображения и текстового эмбеддинга промпта; оценивает сохранение семантики.

\textbf{FID}~\cite{heusel2017fid}~--- расстояние Фреше между распределениями активаций Inception-v3, реализация clean-fid~\cite{parmar2022cleanfid}; 500~реальных изображений ArtBench-10 против 100~сгенерированных. \textit{Approximate FID, reduced protocol.}

\textbf{LPIPS}~\cite{zhang2018lpips}~--- перцептивное сходство на признаках VGG; вспомогательная метрика текстурного соответствия (расстояние).

\subsubsection*{Схема оценки}

Полный протокол даёт на один метод $N_\text{gen}^{(1)} = N_\text{style} \times N_\text{ref} \times N_\text{prompts} = 2 \times 4 \times 100 = 800$~генераций; для сравнения трёх методов (Phase~4)~--- $N_\text{gen}=2\,400$. Дополнительно $1\,000$~генераций для FID ($100$~промптов~$\times 10$~стилей ArtBench, seed~=~42).

На каждый метод: (1)~обучить $\Theta_\text{style}$ на одном стилевом изображении (по прогону на каждое из 4~изображений стиля); (2)~сгенерировать 100~изображений по промптам MS-COCO; (3)~вычислить DINO-style, CLIP-style, CLIP-content на 100~генерациях; (4)~вычислить approximate FID на 1\,000~генерациях против 500~ArtBench-10 соответствующего жанра. Итоговые значения усредняются по 4~стилевым изображениям внутри стиля. Параметры инференса: \texttt{seed=42}, 28~шагов сэмплирования, guidance scale~$3{,}5$.

\subsubsection*{Сравниваемые методы}

В Phase~4 сравниваются четыре метода:
\begin{itemize}\itemsep0.2em
    \item \textbf{B-LoRA-FLUX} (предлагаемый)~--- LoRA на double-stream блоках FLUX.1; конфигурация определяется аблационным исследованием (раздел~\ref{sec:ablations}); при инференсе применяется только $\Theta_\text{style}$.
    \item \textbf{Full-LoRA-FLUX}~--- LoRA на всех блоках FLUX.1 (без блочного разделения); контрольная точка для проверки необходимости селективного выбора блоков.
    \item \textbf{SplitFlux}~\cite{yang2025splitflux}~--- блочное разделение на single-stream блоках; конкурирующий метод внутри FLUX, оценивает относительный вклад выбора типа блоков (DS vs.\ SS).
    \item \textbf{IP-Adapter-FLUX}~\cite{ye2023ipadapter}~--- адаптер изображения без файнтюнинга. На момент проведения экспериментов отсутствовала стабильная публичная реализация, совместимая с FLUX.1-dev; опробованные кандидатные репозитории не обеспечили воспроизводимого пайплайна. Метод не включён в итоговое сравнение; пробел зафиксирован как ограничение работы.
\end{itemize}

В Phase~5 дополнительно проводится кросс-архитектурное сравнение с \textbf{B-LoRA-SDXL}~\cite{frenkel2024blora}: те же 4~стилевых изображения Ван~Гога, унификация разрешения до $512{\times}512$ перед вычислением метрик, фиксированный seed~=~42.

\subsubsection*{Аблационные исследования}
\label{sec:ablations}

Аблационное исследование проводится последовательными фазами на стиле Ван~Гога (одно фиксированное изображение, seed~=~42). Результаты каждой фазы определяют конфигурацию следующей; переход регулируется контрольной точкой GATE~1 (см.~рисунок~\ref{fig:gate1_decision}).

\begin{itemize}\itemsep0.2em
    \item \textbf{Phase~0~--- анализ блоков FLUX.1.} Для каждого из 19~DS и подмножества SS-блоков $\{0, 10, 20, 30, 37\}$ вычисляется оценка $S_b$~\eqref{eq:phase0_score} в режимах style/content injection ($N=200$~пар промптов). Результаты задают приоритетные диапазоны для D-A и кандидатов на $\Theta_\text{content}$ для Phase~1b.

    \item \textbf{Phase~1~--- диагностическая серия D.} Базовая проверка гипотезы: \texttt{e00\_no\_lora\_baseline} (нижняя граница метрик), \texttt{d01} (DS~$[0\text{--}18]$, $r{=}16$, 1\,000~шагов, $\text{lr}{=}5{\times}10^{-5}$, промпт \texttt{"a sks"}), \texttt{d02} (то же при 2\,000~шагов), \texttt{d03} ($r{=}32$, 1\,000~шагов).

    \item \textbf{Phase~1b~--- поиск $\Theta_\text{content}$.} На трёх перекрывающихся диапазонах DS-блоков обучается отдельный LoRA-адаптер ($r{=}16$, остальные гиперпараметры как в \texttt{d01}): ранний $[0\text{--}8]$ (DC-01), поздний $[9\text{--}18]$ (DC-02), центральный $[5\text{--}12]$ (DC-03). При инференсе применяется только данный адаптер; измеряются CLIP-content и DINO-style. Адаптер признаётся содержательным, если CLIP-content возрастает относительно базовой линии, а DINO-style остаётся ограниченным.

    \item \textbf{GATE~1.} По итогам Phase~1 и Phase~1b принимается один из трёх сценариев (A/B/C, см.~раздел~\ref{sec:method}), определяющий план оставшихся фаз.

    \item \textbf{D-A (Phase~2.1)~--- диапазон DS-блоков.} При фиксированном ранге и числе шагов проверяются [0--6], [0--12], [0--18], [6--18].

    \item \textbf{D-B (Phase~2.2)~--- ранг LoRA.} $r \in \{4, 16, 32, 64\}$; $r{=}64$ соответствует конфигурации оригинала~\cite{frenkel2024blora}.

    \item \textbf{D-C (Phase~2.3)~--- число шагов.} $T \in \{500, 1\,000, 2\,000, 4\,000\}$.

    \item \textbf{D-P (Phase~2.4)~--- тренировочный промпт.} \texttt{"a sks"}, \texttt{"a sks painting"}, \texttt{"a [v]"}, \texttt{"a [v] painting in [s] style"}; влияние текстового кондиционирования на разделение стиля и содержания.

    \item \textbf{G (Phase~3)~--- коэффициент применения адаптера.} $\alpha \in \{0{,}3;\, 0{,}5;\, 0{,}7;\, 1{,}0;\, 1{,}5;\, 2{,}0\}$ при финальной конфигурации блоков, ранга, шагов и промпта. По оригиналу~\cite{frenkel2024blora} ожидаемый оптимум~$\approx 0{,}4{-}0{,}5$.

    \item \textbf{Phase~3.2 (только Сценарий~A)~--- раздельные $\alpha_\text{style}$, $\alpha_\text{content}$.} $\alpha_\text{style} \in \{0{,}4;\, 0{,}5;\, 0{,}7;\, 1{,}0\}$ при фиксированном $\alpha_\text{content}{=}1{,}0$ и обратно.

    \item \textbf{Phase~4b~--- анализ ограничений.} Три класса артефактов: \textit{color leakage} (средняя цветовая разница $\Delta E$ в CIELAB по 5~промптам с нейтральным фоном), \textit{background leakage} (DINO-object similarity на патчах переднего плана vs.\ фона по 5~промптам), \textit{complex scenes} (CLIP-content по 5~промптам с $\geq 3$~объектами); 5~генераций на промпт.

    \item \textbf{Phase~5~--- кросс-архитектурное сравнение} с B-LoRA-SDXL по 4~референсам Ван~Гога; метрики: DINO-style, CLIP-style, CLIP-content, FID.

    \item \textbf{Phase~5.2~--- количественное сравнение по протоколу B-LoRA Table~1.} Парный набор DS8 (50~пар при \texttt{seed=42}, по одному изображению на пару); основная метрика~--- DINO~ViT-B/8 сходство между генерацией и стилевым референсом. Поскольку пул отличается от оригинального, абсолютные значения не сопоставимы напрямую с опубликованной таблицей; сравнение методов на идентичном наборе входных данных остаётся корректным.
\end{itemize}

\subsubsection*{Воспроизводимость}

Генерация выполняется скриптом \texttt{scripts/eval/generate\_images.py} с параметрами из Hydra-конфигов (\texttt{configs/eval/generate.yaml}); конфигурации обучения~--- в \texttt{configs/experiments/}. Метрики вычисляются скриптом \texttt{scripts/eval/compute\_metrics.py} и логируются в ClearML~\cite{clearml2024}; прогресс экспериментов фиксируется в \texttt{experiments/plan.md} автоматически.
