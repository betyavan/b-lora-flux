\subsection{Протокол оценки}
\label{sec:eval_protocol}

\subsubsection*{Метрики качества}

Для количественной оценки результатов переноса стиля применяется набор из четырёх
метрик, каждая из которых измеряет отдельный аспект качества генерации.

\textbf{CLIP-style} — косинусное сходство между эмбеддингами CLIP~\cite{radford2021clip}
сгенерированного изображения и стилевого референса. Метрика оценивает степень
соответствия визуального стиля эталону: чем выше значение, тем точнее воспроизведена
художественная манера. Вычисляется по модели CLIP ViT-L/14 на нормализованных векторах.

\textbf{CLIP-content} — косинусное сходство между эмбеддингом CLIP сгенерированного
изображения и текстовым эмбеддингом входного промпта. Метрика оценивает, насколько
хорошо сохраняется семантическое содержание при переносе стиля: высокое значение
означает, что стилизованное изображение соответствует заданному промпту.

\textbf{FID} (\textit{Fréchet Inception Distance})~\cite{heusel2017fid} — расстояние
Фреше между распределениями активаций сети Inception-v3 на реальных и сгенерированных
изображениях. Вычисляется с использованием библиотеки clean-fid~\cite{parmar2022cleanfid}
на выборке из датасета ArtBench-10 (100~изображений на стиль). Низкое значение FID
соответствует высокому фотореализму и близости к целевому стилевому распределению.

\textbf{LPIPS} (\textit{Learned Perceptual Image Patch Similarity}) --- перцептивное
сходство, вычисляемое на основе активаций сети VGG. Используется как вспомогательная
метрика для оценки текстурного соответствия сгенерированного изображения и стилевого
референса.

\subsubsection*{Схема оценки}

Полный протокол включает $N_\text{gen} = N_\text{style} \times N_\text{methods}
\times N_\text{prompts} = 2 \times 4 \times 100 = 800$~генераций для основного
сравнения методов (группа~E, два стиля: Van~Gogh и Monet) и дополнительно
$1\,000$~генераций для оценки FID на подмножестве ArtBench-10
($100~\text{промптов} \times 10~\text{стилей ArtBench}$, seed фиксирован = 42).

Для каждого метода:
\begin{enumerate}
    \item Обучить LoRA-адаптер $\Theta_\text{style}$ на одном стилевом изображении
          (по одному прогону на каждое из 4~изображений стиля).
    \item Сгенерировать изображения по 100~промптам из MS-COCO val2017
          с применением $\Theta_\text{style}$.
    \item Вычислить CLIP-style (сравнение с референсным изображением)
          и CLIP-content (сравнение с текстовым промптом).
    \item Вычислить FID между 1000~сгенерированными изображениями
          и подвыборкой ArtBench-10 соответствующего стиля.
\end{enumerate}

Итоговые значения метрик усредняются по 4~стилевым изображениям внутри каждого стиля
(Van~Gogh, Monet). Все генерации производятся при фиксированном значении seed~=~42,
шагах сэмплирования~=~28 и guidance scale~=~3{,}5.

\subsubsection*{Сравниваемые методы}

В основном эксперименте (группа~E) сравниваются четыре метода:

\begin{itemize}
    \item \textbf{B-LoRA-FLUX} (предлагаемый) — LoRA-адаптер на блоках~30–37
          single-stream FLUX.1; при инференсе применяется только $\Theta_\text{style}$.
    \item \textbf{Full-LoRA-FLUX} — LoRA-адаптер на всех блоках FLUX.1 (без
          блочного разделения); служит для проверки гипотезы о необходимости
          селективного выбора блоков.
    \item \textbf{IP-Adapter-FLUX} — метод на основе адаптера изображения,
          не требующий файнтюна~\cite{ye2023ipadapter}.
    \item \textbf{B-LoRA-SDXL} — оригинальный метод B-LoRA~\cite{frenkel2024blora}
          на базовой модели SDXL; служит для оценки прироста качества
          при переходе к FLUX.1.
\end{itemize}

\subsubsection*{Аблационные исследования}

Помимо основного сравнения, проводятся три серии аблаций (все на стиле Van~Gogh,
по одному фиксированному изображению):

\begin{itemize}
    \item \textbf{Аблация~A: выбор стилевых блоков} — проверяется четыре диапазона
          single-stream блоков: [34–37], [30–37], [24–37], [19–37].
          Цель: эмпирически верифицировать гипотезу о специализации поздних блоков.
    \item \textbf{Аблация~B: ранг LoRA} — $r \in \{4, 8, 16, 32\}$ при
          фиксированных блоках [30–37] и 500~шагах.
    \item \textbf{Аблация~C: число шагов} — $\text{steps} \in \{100, 250, 500, 1000\}$
          при $r{=}16$ и блоках [30–37].
\end{itemize}

По результатам аблаций выбирается конфигурация с наилучшим балансом CLIP-style
и CLIP-content, после чего она применяется в основном эксперименте (группа~E)
и финальной проверке на стиле Monet (группа~F).

\subsubsection*{Воспроизводимость}

Генерация изображений производится скриптом \texttt{scripts/eval/generate\_images.py}
с параметрами, заданными через Hydra-конфиг (\texttt{configs/eval/generate.yaml}).
Конфигурации обучения для каждого эксперимента хранятся в \texttt{configs/experiments/}.
Все метрики вычисляются скриптами \texttt{scripts/eval/compute\_metrics.py} и
логируются в ClearML~\cite{clearml2024}. Прогресс экспериментов отслеживается
в \texttt{experiments/plan.md}, обновляемом скриптом \texttt{scripts/update\_exp\_plan.py}.
