\subsection{Используемые данные}
\label{sec:datasets}

Метод B-LoRA относится к категории однофотографических подходов: для файнтюна
достаточно одного изображения-референса, после чего модель способна воспроизводить
заданный стиль в новых генерациях. Тем не менее экспериментальная оценка требует
трёх категорий данных: обучающих образцов стилей, референсного распределения для
вычисления метрик качества и контентных изображений для оценки семантического
соответствия.

\subsubsection*{Эталонные изображения стилей}

Для демонстрации и тонкой настройки B-LoRA отобрано 8~изображений, представляющих
два художественных стиля: импрессионизм Клода Моне и постимпрессионизм Винсента
ван~Гога. Изображения получены с платформы WikiArt и выбраны вручную по критериям
стилистической выраженности и цветовой характерности.

Для ван~Гога задействованы: «Звёздная ночь» (1889), «Красные виноградники» (1888),
«Подсолнухи» (1888), «Автопортрет с перевязанным ухом» (1889). Для Моне: «Стог сена
в~Живерни» (1886), «Водяные лилии --- вечерний эффект» (1899), «Мост Ватерлоо» (1903),
«Мадам Моне с ребёнком» (ок.~1875). При каждом запуске файнтюна используется одно
изображение; набор из~четырёх позволяет оценить вариативность результатов внутри
одного стиля.

Выбор обусловлен стилистической контрастностью: импрессионизм Моне характеризуется
размытыми контурами и пастельной палитрой, тогда как манера ван~Гога отличается
выраженной вихреобразной фактурой и насыщенным цветом. Это позволяет проверить,
насколько стилевые блоки FLUX фиксируют различные по природе текстурные характеристики.

\subsubsection*{ArtBench-10}

Для количественной оценки качества переноса стиля применяется датасет
ArtBench-10~\cite{liao2022artbench}, содержащий около 60\,000~изображений,
равномерно распределённых по 10~художественным стилям: импрессионизм,
постимпрессионизм, экспрессионизм, кубизм, абстрактный экспрессионизм и~др.
Для вычисления FID используется подвыборка по 500~изображений на жанр
(\textit{reduced protocol}): генерированные изображения сравниваются
с~реальными работами соответствующего стиля, что~даёт оценку соответствия
распределения признаков. Ввиду ограниченного объёма референсной выборки метрика
применяется для сравнения конфигураций между собой, но не для прямого
сопоставления с опубликованными значениями FID.
Источник: HuggingFace Hub (\texttt{Doub7e/ArtBench-10}), лицензия CC-BY.

\subsubsection*{MS-COCO val2017}

Для оценки сохранения семантического содержания используется подвыборка из
100~изображений валидационного раздела MS-COCO 2017~\cite{lin2014coco}.
Разнообразные сцены без выраженного стилевого смещения обеспечивают нейтральный
контентный фон для измерения CLIP-сходства между исходным и стилизованным
изображениями. Источник: \url{cocodataset.org}, лицензия CC-BY. Протокол оценки
с~учётом числа стилей и методов сравнения подробнее описан в~разделе~\ref{sec:eval_protocol}.

\subsubsection*{DreamBooth: предметные фотосессии (DS5)}

Для формирования пула контентных изображений задействован датасет Google
DreamBooth~\cite{ruiz2023dreambooth}: репозиторий \texttt{google/dreambooth}
предоставляет предметные папки с несколькими снимками одного объекта (лицензия
Apache~2.0). В экспериментах используются 8~предметов: \textbf{dog}
($\approx$30~изображений~--- основной объект для диагностических экспериментов
Phase~1b) и 7~дополнительных (\textit{cat}, \textit{backpack}, \textit{bowl},
\textit{can}, \textit{clock}, \textit{vase}, \textit{bear}); в каждой папке
содержится около 25~изображений объекта, что в сумме даёт примерно
$8\times 25 \approx 200$~изображений. Совокупно они образуют пул
контентных изображений для количественного сравнения методов (Phase~5.2).
Подпись при файнтюне: \texttt{"a~sks~<class>"}.

\subsubsection*{Набор для парного сравнения (DS8)}

Количественное сравнение методов воспроизводит протокол раздела~5.1 оригинальной
статьи~\cite{frenkel2024blora} в виде \textit{reduced protocol}, принятого для ВКР.
Пул контента~--- $N_c{=}8$ объектов из DS5, пул стилей~--- $N_s{=}8$ референсов
из DS1 (ван~Гог~$\times$~4, Моне~$\times$~4). Из декартова произведения
$8{\times}8{=}64$~возможных пар семплируется $\min(50,\;N_c{\cdot}N_s){=}50$~пар
при \texttt{seed=42}; манифест фиксируется в
\texttt{experiments/data/b\_lora\_eval\_pairs.json} и версионируется через DVC.
Основная метрика~--- среднее косинусное сходство DINO ViT-B/8 между сгенерированным
изображением и стилевым референсом, аналогично Table~1 оригинала. Поскольку
абсолютные значения метрики зависят от состава пула, сравнение с числами из
оригинальной статьи носит ориентировочный характер; относительные различия между
методами остаются корректными.

\subsubsection*{Версионирование данных}

Все данные версионированы с помощью DVC (Data Version Control): файлы \texttt{.dvc}
фиксируют хэш каждого изображения и хранятся в репозитории вместе с кодом, тогда
как фактические данные вынесены в удалённое хранилище. Воспроизведение идентичного
набора данных обеспечивается командой \texttt{dvc pull} после клонирования репозитория.
